## Workflow Decision

- Use QIIME 2 when starting from demultiplexed FASTQ files, manifests, `.qza`/`.qzv` artifacts, or when provenance needs to be preserved.
- Use DADA2 in R when the project is R-first and FASTQ-to-ASV processing should stay close to the downstream analysis.
- Use phyloseq, vegan, microeco, ANCOMBC, MaAsLin2, DESeq2, or custom Wilcoxon workflows when the inputs are already a feature table, taxonomy table, and metadata.
- Preserve the project's existing analysis route unless there is a clear reason to convert formats or tools.

## Minimum Inputs

Confirm these before writing analysis code:

- Raw reads or feature table path.
- Metadata path and sample ID column.
- Taxonomy table or classifier database.
- Representative sequences and phylogenetic tree if UniFrac or tree-aware methods are needed.
- Main grouping variables, batch variables, paired/repeated-measure variables, and covariates.
- Target taxonomic levels: ASV, genus, family, phylum, or project-specific ranks.
- Output directory and expected figure/table formats.

## QIIME 2 Pipeline

For a CLI-first QIIME 2 workflow:

1. Define paths and parameters at the top: manifest, metadata, classifier, thread count, date tag, and output root.
2. Use strict shell behavior such as `set -euo pipefail`.
3. Add timestamped logging, required-file checks, and directory creation helpers.
4. Import paired-end reads as `SampleData[PairedEndSequencesWithQuality]` with the correct manifest format and Phred encoding.
5. Generate a demux summary and inspect quality before choosing DADA2 trim/truncation values.
6. Run `qiime dada2 denoise-paired` or the project's chosen denoiser with explicit parameters.
7. Assign taxonomy with a classifier matching the amplicon region and database version.
8. Export the feature table, taxonomy, representative sequences, and optional stats.
9. Convert BIOM to TSV if continuing in R.

Good numbered artifact directories:

- `1.import_data/`
- `2.dada2/`
- `3.taxonomy/`
- `4.exports/`

Do not copy trim/truncation parameters between projects without inspecting read quality.

## R Downstream Workflow

For an R-centered downstream workflow, standardize exported files into a clean analysis directory:

- `otutab.csv`: ASV/OTU counts with features as rows and samples as columns.
- `taxonomy.csv`: taxonomy table with a feature ID column.
- `metadata.csv`: metadata with sample IDs matching feature-table columns.
- `rep-seqs.fasta`: representative sequences if available.
- `asv_id_conversion.csv`: mapping from original sequence IDs to simplified ASV IDs when IDs are renamed.
- `otutab_flattened.csv`: rarefied count table when rarefaction is needed.
- `Relative_otutab.csv`: relative abundance table.

Recommended R packages:

- Data handling: `tidyverse`, `data.table`, `readr`.
- Diversity and ordination: `vegan`, `ape`, `picante`.
- Sequence handling: `Biostrings`.
- Microbiome objects and biomarker methods: `phyloseq`, `microeco`.
- Differential abundance and heatmaps: `DESeq2`, `pheatmap`, `RColorBrewer`.

Common cleanup steps:

1. Parse taxonomy into `Phylum`, `Class`, `Order`, `Family`, `Genus`, and `Species`; fill missing values as `Unclassified` or `Unknown`.
2. Filter low-quality or unwanted ASVs with explicit thresholds and keep taxonomy synchronized with the table.
3. Check feature and sample alignment with `stopifnot()` or equivalent assertions.
4. Optionally rename filtered features to `ASV1`, `ASV2`, ... and save a conversion table.
5. If representative sequences exist, rename FASTA records using the same conversion table.
6. Create relative abundance by dividing each sample column by its total reads and verify column sums.

## Diversity Analysis

Alpha diversity:

- Use rarefied counts when the chosen alpha workflow assumes equal depths.
- Record the rarefaction depth and random seed.
- Shannon is a good default display metric; add observed richness, Simpson, or Chao1 when biologically useful.
- For two-group comparisons, Wilcoxon tests are common; for multi-factor or paired designs, use models that match the study design.

Beta diversity:

- Bray-Curtis is a good default for abundance-based beta diversity.
- Use Jaccard for presence/absence and UniFrac only when a reliable phylogenetic tree is available.
- For PCoA, report the first two axis percentages.
- For PERMANOVA, report the distance metric, formula, permutations, R2, p-value, and any strata/blocking.
- Check or discuss dispersion with `betadisper()` when interpreting PERMANOVA.
- Use CAP/db-RDA with `capscale()` when constrained ordination is desired.

Always align the abundance matrix and metadata by sample ID before distance calculations.

## Taxonomic Composition

Useful defaults:

- Aggregate ASV counts to family or genus before group summaries.
- Convert to relative abundance per sample.
- Keep the top N taxa by total abundance and collapse the rest into `Others`.
- Use top 10 families or top 15 genera as a starting point.
- Summarize by project grouping variables before plotting stacked bars.
- State whether bars show sample-level abundances or group means.

## Differential Abundance And Biomarkers

Choose the method based on the question and data:

- LEfSe-style biomarker discovery: use for exploratory group-enriched taxa and report LDA thresholds.
- Wilcoxon pairwise tests on relative abundance: useful for exploratory ASV/taxon screens; report p-value/FDR rules and pseudo-counts used for log2 fold change.
- DESeq2: use raw counts with explicit filtering and a count-model rationale.
- ANCOM-BC or MaAsLin2: prefer for compositional/covariate-aware final analyses when appropriate.

Common comparison patterns:

- Within each treatment, compare control genotype/group against each mutant or condition.
- Within each genotype/group, compare treatment versus control.
- For combined designs, create a `Group` variable such as `paste(Genotype, Treatment, sep = "_")`, then use explicit contrast pairs.

Heatmap pattern for differential ASVs:

1. Collect significant features from pairwise comparisons.
2. Use relative abundance to compute group means for display.
3. Apply row-wise z-score scaling.
4. Fix column order to match the experimental design.
5. Cluster rows, optionally cut the tree into a small number of clusters.
6. Label rows with a taxonomic rank plus feature ID when helpful.
7. Export both the heatmap and a cluster-info table.

## Output Layout

A reproducible project can use:

- `data/filter_otu/`
- `data/renamed_otu/`
- `results/alpha/`
- `results/beta/` or `results/PCOA/`
- `results/composition/`
- `results/LEfSe/`
- `results/differential_abundance/`
- `logs/`

Save machine-readable tables next to figures whenever practical.

## Pitfalls

- Sample IDs often differ by dashes, underscores, lane suffixes, or file extensions.
- Taxonomy column positions vary across exports; check column names before assuming rank positions.
- Relative abundance barplots are descriptive, not differential abundance evidence.
- Rarefaction can be useful for diversity but should not silently replace count-aware or compositional methods.
- Exploratory p-value thresholds should not be presented as confirmatory without caveats.
- Batch, site, run, subject, cage, timepoint, and treatment effects can be confounded; inspect metadata before choosing formulas.
